# Complete Metadata Management System Guide

## Обзор

Система управления метаданными позволяет полностью контролировать все атрибуты слайдов через веб-интерфейс без необходимости изменения кода.

## Основные возможности

### 1. Динамические метаданные (Dynamic Metadata)

Управляются через Settings → Dynamic Entities

- **Solution Areas** (Направления решений)
  - Marketing, Sales, Engineering, Design, Consulting, Management, HR, Finance
  - Одиночный выбор для каждого слайда
  
- **Domains** (Отрасли/Индустрии)  
  - Automotive, FinTech, Retail, Healthcare, Education и др.
  - Одиночный выбор для каждого слайда

### 2. Расширенные метаданные (Extended Metadata)

Управляются через Settings → Extended Metadata

- **Products** (Продукты)
  - IT Services, Application Support, IT Management, Security, TCO Optimisation
  - Множественный выбор
  
- **User Types** (Типы пользователей)
  - Client, Employee, Partner
  - Множественный выбор
  
- **Components** (Компоненты)
  - Table, Chart, Logo, Roadmap, Team Content и др.
  - Множественный выбор
  
- **Integrations** (Интеграции)
  - Jira, Slack, CRM, Google
  - Множественный выбор

### 3. Категории (Categories)

Управляются через Settings → Categories

- Иерархическая структура (до 3 уровней вложенности)
- Используются для организации библиотеки слайдов
- Каждый слайд может быть только в одной категории

### 4. Теги (Tags)

Управляются через Settings → Tags

- Автоматические теги (генерируются из текста слайдов)
- Ручные теги (добавляются пользователями)
- Цветовая маркировка

### 5. Статические значения (Static Values)

Управляются через Settings → Static Values

- **Statuses** (Статусы)
  - Draft, In Review, Approved, Archived
  
- **Formats** (Форматы)
  - Vertical, Horizontal
  
- **Languages** (Языки)
  - English, French, German, Multilingual
  
- **Regions** (Регионы)
  - Global, EMEA, North America, Asia-Pacific, Latin America

## Использование в библиотеке

### Отображение метаданных
- Все метаданные отображаются на карточках слайдов
- Категории показаны в дереве слева
- Метаданные показывают реальные названия, а не коды

### Поиск по метаданным
Поиск работает по всем полям:
- Основные поля (название, описание, извлеченный текст)
- Теги
- Solution Areas, Domains
- Products, Components, Integrations  
- User Types
- Автор
- Годы проекта
- Case Study статус

### Фильтрация
- По категориям через дерево категорий
- По всем остальным метаданным через поиск

## Редактирование метаданных слайдов

### Одиночное редактирование
1. Нажмите на слайд в библиотеке
2. В модальном окне нажмите "Edit Details"
3. Измените нужные метаданные
4. Сохраните изменения

### Массовое редактирование
1. Перейдите в раздел Moderation
2. Выберите слайды галочками
3. Нажмите "Edit Metadata"
4. Измените метаданные для всех выбранных слайдов

## Управление метаданными

### Добавление новых значений
1. Перейдите в Settings
2. Выберите нужную вкладку
3. Введите код и название
4. Нажмите кнопку добавления

### Редактирование существующих
1. Найдите нужное значение
2. Нажмите иконку карандаша
3. Измените название
4. Нажмите галочку для сохранения

### Удаление
1. Нажмите иконку корзины
2. Подтвердите удаление
3. Система покажет количество связанных слайдов

## Автоматическая синхронизация

- Изменения в Settings мгновенно отражаются во всем приложении
- Фоновая синхронизация каждые 30 секунд
- При редактировании происходит принудительное обновление

## Техническая информация

### База данных
- PostgreSQL с Prisma ORM
- Все метаданные хранятся в отдельных таблицах
- Связи many-to-many для множественного выбора

### API endpoints
- `/api/metadata` - получение всех метаданных
- `/api/solution-areas` - управление Solution Areas
- `/api/domains` - управление Domains
- `/api/products` - управление Products
- `/api/user-types` - управление User Types
- `/api/components` - управление Components
- `/api/integrations` - управление Integrations
- `/api/categories` - управление Categories
- `/api/tags` - управление Tags
- `/api/statuses` - управление Statuses
- `/api/formats` - управление Formats
- `/api/languages` - управление Languages
- `/api/regions` - управление Regions

### Контекст метаданных
- `MetadataContext` предоставляет доступ ко всем метаданным
- Автоматическое кэширование для производительности
- Хук `useMetadataContext()` для использования в компонентах
