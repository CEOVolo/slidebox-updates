# Настройка Figma Access Token

## Проблема
Без Figma Access Token плагин может создавать только изображения слайдов вместо полностью редактируемых элементов.

## Решение

### 1. Получите Figma Access Token

1. Войдите в свой аккаунт Figma
2. Перейдите в настройки: https://www.figma.com/settings
3. Прокрутите вниз до раздела "Personal access tokens"
4. Нажмите "Generate new token"
5. Дайте токену описание (например, "SlideDeck Plugin")
6. Скопируйте токен (он показывается только один раз!)

### 2. Создайте файл .env.local

В корне проекта создайте файл `.env.local` со следующим содержимым:

```
FIGMA_ACCESS_TOKEN=ваш_токен_здесь
DATABASE_URL="file:./prisma/dev.db"
```

Замените `ваш_токен_здесь` на скопированный токен.

### 3. Перезапустите сервер

```bash
npm run dev
```

## Что это даёт?

С настроенным токеном API сможет:
- Получать полную структуру узлов (тексты, формы, стили)
- Воссоздавать редактируемые элементы, а не только изображения
- Работать с реальными файлами Figma

## Проверка

После настройки в логах API вы должны увидеть:
- `Найден узел [nodeId]: { type: 'FRAME', hasChildren: true, ... }`
- В плагине: `Есть структура node: true`

## Безопасность

- НЕ коммитьте `.env.local` в git (он уже в .gitignore)
- Токен даёт read-only доступ к вашим файлам Figma
- Можно отозвать токен в любой момент в настройках Figma
